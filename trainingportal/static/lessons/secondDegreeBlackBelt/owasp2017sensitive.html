About the vulnerabilities: Cryptographic Failure

Scenario #3: The password database uses unsalted or simple hashes to store everyone's passwords. A file upload flaw allows an attacker to retrieve the password database. All the unsalted hashes can be exposed with a rainbow table of pre-calculated hashes. Hashes generated by simple or fast hash functions may be cracked by GPUs, even if they were salted.

From OWASP Top 10 2021 A02 - Cryptographic Failures
Challenge Tips and Tricks

Along with the 'secret' file you will find many other files in the bucket including the html for a chat application that the hackers use. The users file for the application can be found in the same bucket and the passwords are stored as unsalted SHA-1 hashes.
The users of the system have strong passwords but the test account not so much. Crack the test account password and login into the chat to pass this challenge.
