The purpose of this challenge is to demonstrate the MITRE Top 25 programming flaw: 'Improper Authentication'.

When an actor claims to have a given identity, the product does not prove or insufficiently proves that the claim is correct.

From MITRE CWE 287

About the Challenge

The developer of the vulnerable application has implemented improper authentication that can be easily bypassed. The application properly authenticates a user's credentials but then assigns a predictable value to the user's browser local storage or cookies to represent the user's session. See if you can become authenticated without providing valid credentials.
